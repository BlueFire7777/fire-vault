### 3-3. exit code (F300 hardening + Codex 補強)

| code | 意味 |
|---|---|
| 0 | success (events_total >= 50) |
| 1 | error (DB error / fetch error / batch_replay 例外 / **error_msg 非 None**) |

★ Codex 連鎖 1 件目検出による補強:
`BatchReplayResult.error_msg` 非 None は events>=50 でも exit 1 で扱う。

ケース (interrupted/incomplete batch) を OpenClaw/launchd に異常伝播、
偽 PASS = 利確/損切指示永久欠落リスク回避 (F300 hardening 同型)。

## 4. Codex pre-commit 履歴 (連鎖 1 件)

### 4-1. 連鎖 1 件目 (修正対応済)

```
CRITICAL: scripts/jobs/run_backtest_seeding.py ignores
BatchReplayResult.error_msg after runner.run(). Existing
BatchReplayRunner.run() can catch KeyboardInterrupt or internal non-SQLite
exceptions and return a result with error_msg instead of raising; this
wrapper then decides success/partial only from result.n_events, so a fatal
interrupted/incomplete batch can exit 0 if n_events >= 50.
```

修正: `result.error_msg` 非 None で `exit 1 + sys.stderr` を events 判定の
**前** に挿入。test 1 件追加 (`test_error_when_result_has_error_msg`)。
