---

## 1. 既知失敗テスト

**テスト名**:

```
tests/risk/test_execution_gate.py::TestF115Integration::test_f115_decide_orders_rejected_via_gate
```

**現状 reason**:

```
F142 R-17-02 opening restriction: R-17-02 寄り直後制限:
現在 09:00:00 JST、09:05:00 まで観察優先
```

**期待 reason**:

```
F140 gate failed
```

**原因**:
- テスト `as_of_dt = 2026-05-02 00:00:00 UTC = 09:00:00 JST`
- F142 R-17-02 寄り直後制限 (09:00-09:05) が F140 gate より先発火
- F140 gate のチェックに到達せず、テスト assertion FAIL

---

## 2. 解消候補

### 案 a: テスト時刻のみ修正

`as_of_dt` を F142 範囲外 (例: `2026-05-02 00:10:00 UTC = 09:10:00 JST`) に
変更し、F140 gate に到達させる。最小変更だが、F142 と F140 の優先順位は
そのまま。

### 案 b: F140-F142 優先順位設計の見直し

既存実装では F142 R-17-02 が F140 gate より先発火している。本来 F140 gate
(板厚 / スプレッド / 出来高 / 部分約定 / 値飛び / 値嵩) は F142 R-17-02
(時刻ベース) と独立してチェックされるべきか、設計判断要。

### 推奨

着手時に Fujiwara 判断。短期は案 a、長期は案 b 検討。
