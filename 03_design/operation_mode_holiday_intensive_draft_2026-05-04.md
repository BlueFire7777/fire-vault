## 1. 起票背景

FIRE 要件書 v3.3 は **兼業前提**で設計されている (R-01-08 半自動主軸、第 33 章
運用負荷上限 1 日 30 分以内、第 38 章 [休止][旅行][兼業] の 3 モード)。
しかし以下の運用シーンに対する戦略適応が想定外:

- 土日休場日は不要 (運用なし)
- **平日でも有給/在宅日に Mac mini 前で張り付ける**機会
- M+1 以降 Stage 3 が安定し、リスク許容度が拡大した時の積極攻撃機会

兼業時のロット倍数延長やエントリー回数倍増では「**質的な戦略変化**」が
扱えない (スキャル / デイトレ系パターンが回らない、5 分 tick 評価では
スキャル不可能)。

→ **休日専業モード**を新たな運用モードとして追加する設計議論。

## 2. 確定方針 4 点 (Fujiwara 確認済 2026-05-04)

### 方針 1: 本質は「戦略の質的変化」

兼業モードの単純倍数延長 (ロット 2 倍 / エントリー回数 2 倍 / 保有時間 2 倍) ではなく、
**スキャルピング / 短期デイトレ戦略を解放する**。
- 兼業: スイング主軸、デイトレ補助、スキャル不可
- 休日専業: スキャル + デイトレ主軸、スイング併存

### 方針 2: 切替方法は手動

- Fujiwara が朝 (9:00 前) にフラグを立てる
- 当日有効、翌日リセット (デフォルトの兼業モードに自動復帰)
- 自動切替は Phase 1 では実装しない (R-13-08 自動反映禁止に整合)
- フラグは ~/fire/runtime/mode_state.json 等に永続化 (案 1、後述 § 9)

### 方針 3: アグレッシブ度 (兼業比)

倍数比 (兼業 → 休日専業):
- **回数 2.5x** (1 日 5〜8 → 30〜40)
- **保有時間 3x** (兼業の通常保有 30 分 → 休日専業 90 分、ただし上限 2 時間)
- **リスク 2x** (1 日累積 2.0% → 4.0%)

これらは「単純倍数」ではなく、**スキャル/デイトレ戦略でしか成立しない
パラメータ群** (詳細パラメータ表は § 4 参照)。

### 方針 4: 着手時期は Stage 3 安定後 (M+1 以降)

Stage 3 で兼業モードが順調に動き続けることを前提に、その上で休日専業モードを
追加する。Stage 3 移行直後の M=0 / M=0+1 では着手しない (R-39-02/03 半減ルール
と整合、リスク拡大は時期尚早)。

## 4. 休日専業モードのパラメータ詳細 (アグレッシブ版)

| 項目 | 兼業 (デフォルト) | 休日専業 |
|---|---|---|
| 同時保有ポジション数上限 | 3 | **8〜10** |
| 1 日エントリー回数上限 | 5〜8 | **30〜40** |
| 主要戦略 | スイング | **スキャル + デイトレ** |
| 1 トレード最大リスク (資金比) | 0.5〜1.0% | 0.2〜0.4% (個別は減、件数増) |
| 1 日最大リスク累積 | 2.0% | **4.0%** |
| エントリー閾値 (F032 score) | 厳しめ (≥ 0.65 標準) | **大幅緩和 (-30%、≥ 0.45 想定)** |
| 監視頻度 (tick 評価間隔) | 5 分 | **10 秒〜30 秒** |
| LINE 通知頻度 | 重要のみ | **全イベント** |
| 損切りライン | -2% | -0.5% 〜 -1% (タイト) |
| 利確ライン | +5% | +0.5% 〜 +2% (段階的) |
| 保有時間上限 | なし | **最大 2 時間** |
| 強制クローズ時刻 | 15:10 | **同 (R-20-03 準拠、変更なし)** |

**設計意図**:
- 個別リスクを減らし件数を増やす → エッジが小さくても累積で利益を狙う
- 短時間保有 + タイト損切で持ち越しリスク回避
- F032 score 閾値緩和は「件数を取るため」、ただし下方互換でなく **別パラメータセット**
  (兼業モードの閾値は変更しない)
- 監視頻度 10〜30 秒は Mac mini 前張り付き前提、LINE 全通知も同前提

---

## 5. リスク管理ガード (連敗 → 兼業モード強制降格)

休日専業モード稼働中に以下のいずれかが発生したら、**当日内で兼業モードに
強制降格**する:

1. **連敗 3 回**: 直近 3 件の closed positions がすべて label='failure'
2. **DD -2% 到達**: 当日累積損益 ≤ -2% (兼業 1 日上限の 1.0 倍に達した時点)
3. **+5% 利益達成**: 当日累積利益 ≥ +5% (利確逃げ、無理な追加攻撃を避ける)

→ 強制降格後はその日中に休日専業モードに戻れない (翌朝 Fujiwara 再判断)。

## 6. 切替運用ルール

- **タイミング**: 朝 9:00 前 (= zaraba 開始前) に Fujiwara がフラグ立て
- **方法**: ~/fire/runtime/mode_state.json を更新 (CLI コマンド or 手動編集、
  詳細は F279 Phase 1 で確定)
- **9:00 以降ロールバック禁止**: 当日中に「やはり兼業に戻したい」とは
  できない (中途半端なリスクを生むため、§ 5 の強制降格のみが当日切替経路)
- **翌日リセット**: 5:00 (JST) の launchd cron で自動的に兼業モードに戻る

## 7. スキャル / デイトレ戦略の実装範囲

休日専業モードを実用化するには、**新規パターン定義**が必要:

| カテゴリ | 既存 (兼業) | 新規 (休日専業) |
|---|---|---|
| material 系 | material_initial / earnings_upside / etc | (流用) |
| 価格アクション | breakout / pullback (デイ向け) | **scalp_VWAP_bounce / scalp_breakout_5min** |
| 時間軸 | AM (寄付) / PM (大引け) | **9:00-9:30 / 9:30-10:00 / 10:00-11:30** (粒度細かく) |
| 流動性 | highliq | highliq + **superliq** (TOPIX Core30 のみ) |

→ パターン Store の Layer 設計に影響あり。詳細は § 8。

## 8. パターン Store 階層整合 (Layer 設計判断)

要件書第 37 章 v3.4:
- Layer 1: Pattern Archive (全履歴 100+)
- Layer 2: Valid Pattern Library (20-100)
- Layer 3: Active Priority Set (3-15)
- Layer 4: Death Note / Rehab

**休日専業モード追加で生じる質問**:
1. スキャル/デイトレ用 Active Priority Set は **兼業の Active と分けるか?**
2. 兼業モード時にスキャルパターンが SimilarityEngine.search() に hit すると
   ノイズになるか?
3. mode_state によって Active Priority Set を切り替える設計は妥当か?

**設計判断 (草案、F279 Phase 1 で確定)**:
- Active Priority Set を **mode 別に持つ** (兼業 active 5-8 件 + 休日専業 active 5-8 件)
- SimilarityEngine.search() に `mode` 引数を追加、現在の mode に応じてフィルタ
- 要件書第 37 章 v3.4 → v3.5 改訂が必要 (Layer 3 を mode 別に再定義)

→ **要件書 R-37-XX 改訂**を F279 Phase 1 のスコープに含める。

## 10. Claude.ai 側からの懸念点 3 つ

### 懸念 1: 再現性優位戦略との相性

FIRE のエッジは「**再現性優位**」(第 26 章)、Pattern Store 蓄積による高再現性
パターンの照合がコア。スキャルパターンは:
- 1 トレードあたりエッジが小さい (0.5〜2% 利確)
- 件数が多い (30〜40 件/日) 分、ノイズ混入率も高い
- F032 SimilarityEngine の閾値緩和でさらにノイズ増

→ **再現性優位の質的変化**が起こる可能性。検証必要 (§ 11 仮説 X)。

### 懸念 2: 心理負荷

休日専業モード稼働中は LINE 全通知 + 10 秒 tick + 8〜10 同時保有 = 高心理負荷。
Fujiwara のメンタル状態が低下すれば DD 拡大、§ 5 強制降格頻発のリスク。

→ M+1 以降に **連続 1 ヶ月** で休日専業を回せるか、Phase 2 で実証必要。

### 懸念 3: 生活習慣リスク

休日も Mac mini 前に 6〜8 時間張り付くと、本業以外の時間配分が不健全化する
リスク (= 「兼業の安全感」が崩れる)。F279 Phase 1 で **休日専業稼働日数の
月次上限** (例: 月 4 日まで = 週末スポット投入のみ) を含めるか検討。
