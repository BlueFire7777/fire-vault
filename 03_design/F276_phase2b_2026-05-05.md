## 4. circular dependency 解消経路の検証

### 4-1. 経路全体図

```
J-Quants V2 (subscription)
    ↓ market_data/client.py (get_indices_topix / get_indices)
    ↓ market_data/index_fetcher.py (CLI、0/1/2 exit code、Fujiwara cron)
index_data テーブル (新規 schema、UNIQUE date+symbol、idempotent)
    ↓ features/regime.py build_snapshot_from_db (新規 helper)
    ↓ MarketSnapshot (nikkei_change_pct + topix_change_pct)
RegimeFeatureCollector.collect() (既実装、snapshot 引数で稼働)
    ↓
regime feature 実値化:
  - market_regime: "レンジ" (default) → "強地合い"/"弱地合い"/"決算相場"等
  - nikkei_direction: None (default) → -1.0〜1.0 (実値)
  - topix_direction: None (default) → -1.0〜1.0 (実値)
    ↓
ReproducibilityEngine.evaluate() で regime_fit が default 0.5 → 実値 0.7-0.9
    ↓
score 0.5787 → 0.5787 + α (F104 寄与 +0.02〜+0.04 予測、F276 todo §B)
```

### 4-2. test で経路稼働確認

`TestCollectIntegration::test_collect_with_index_db_snapshot` で実証:
- index_data に TOPIX (change_pct=2.5) + N225 (change_pct=2.5) を seed
- build_snapshot_from_db で MarketSnapshot 構築
- RegimeFeatureCollector.collect() に snapshot を渡す
- result["regime_label"] == "強地合い" (default "レンジ" から変化)
- features["nikkei_direction"] == 1.0 (default None から実値)

### 4-3. 線形外挿楽観視回避 (§6-7)

regime_fit_factor が default 0.5 から実値 0.7-0.9 に変化する経路は test で
稼働確認済だが、**実 Run a での events_total 実測検証は Phase 2-B 完了後の
段取り** (本部指示書通り、F276 Phase 2 着手指示文 §「Phase 2-B 完了後の
1 day Run a 実測」)。

実測値次第で:
- events_total ≥ 50 達成 → Phase 2-C (Layer 3 拡大、運用のみ) で完結
- events_total < 50 → Phase 2-C に SCORE_THRESHOLD_EXECUTE 微調整追加
                       (Fujiwara 承認、F275 line 161 前例、案 a 部分採用)
