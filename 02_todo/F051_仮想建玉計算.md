# F051: 仮想建玉 / 平均建値 / 評価損益 / 実現損益 / 余力管理

## 概要

F050 PaperLiveRunner.tick() で発生する仮想イベント (virtual_entry / virtual_tp / virtual_sl /
force_close) を処理し、**仮想建玉** と **仮想口座** の状態を更新する `PositionTracker` /
`AccountTracker` を実装。R-19-04 (仮想執行対象) を満たす。

## 実装内容

### 主要モジュール

- `simulation/paper_live/position.py` (新規): `PositionTracker`
  - `apply_event(event)`: F050 イベント種別で分岐 (entry / close / no-op)
  - 新規エントリー / 買い増し (加重平均) / 全決済 / 部分決済を実装
  - `_calc_realized_pnl(side, avg, exit, qty)`: long/short 対応の実現損益
- `simulation/paper_live/account.py` (新規): `AccountTracker` (事後集計型)
  - `init_account(initial_capital)`: 口座初期化
  - `refresh()`: 現在の建玉から `n_positions_*` / `realized_pnl_total` /
    `current_capital` / `lot_used` を再計算
  - `update_unrealized(prices)`: 現在価格辞書から含み損益を再計算
  - `halt(reason)` / `resume()`: 仮想停止フラグ管理
- `simulation/paper_live/models.py` (拡張): `VirtualPosition` / `VirtualAccount` /
  `PositionStatus` Enum / `CloseReason` Enum 追加
- `simulation/paper_live/runner.py` (最小拡張):
  - `start_session(initial_capital=1000万円)`: Tracker 初期化 + `init_account()`
  - `tick()`: 各 event を `position_tracker.apply_event()` に流し、events があれば
    `account_tracker.refresh()`
  - `end_session()`: Tracker クリア
- `simulation/paper_live/report.py` (拡張): `generate_markdown_report` に
  「仮想口座サマリ (F051)」セクションを追加 (account がある場合のみ表示)
- `scripts/setup/migrate_paper_live_positions.py` (新規):
  `paper_live_positions` / `paper_live_account` 新規テーブル + 3 インデックス

### キーポイント

- **PositionTracker**: SQLite で建玉を管理。同銘柄・同サイドの open ポジションが
  あれば買い増し (加重平均更新)、無ければ新規 INSERT。クローズは TP/SL/FORCE_CLOSE
  で `close_reason` を切り替え。部分決済は既存数量を減らして決済分を別レコードで closed
- **AccountTracker は事後集計型**: tick の events 反映後に refresh() で再計算する設計。
  含み損益は現在価格が必要なので別メソッド (update_unrealized) で分離
- **既存 F050 24 PASS 非破壊**: tick() 拡張は events=[] では tracker が no-op で完走
- **F050 fixture に F051 migration を追加**: runner が account に依存するため、
  fixture も更新 (これがないと既存 24 PASS が壊れる)
- **FK 制約**: paper_live_positions / paper_live_account は paper_live_runs を参照。
  テストの fixture では parent run を先に INSERT する必要あり
