# F131: M=0 初月半減ルール (M=0/M+1/M+2 段階的リスク調整)

## 概要

第 5 章 R-05-03/05 + 第 39 章 R-39-02/03/04/05 のステージ別リスク係数を実装。
F130 `calculate_allowable_loss` に `stage` 引数を追加し、`OperationalStage` enum
(PAPER_LIVE / M0 / M1 / M2_PLUS) で運用ステージごとに倍率を適用。

## スコープ

- `OperationalStage` enum (PAPER_LIVE / M0 / M1 / M2_PLUS、str 継承)
- `STAGE_MULTIPLIER` テーブル (1.0 / 0.5 / 0.75 / 1.0)
- `calculate_allowable_loss` / `get_loss_rate` に `stage` 引数追加 (デフォルト
  PAPER_LIVE で非破壊)
- `get_operational_stage_from_env()` で `.env` の `OPERATIONAL_STAGE` 読み込み
- 不正値・未設定はログ警告して PAPER_LIVE フォールバック (例外で止めない)

## 確定数値 (要件書整合)

| ステージ | 倍率 | デイトレ通常 | スイング通常 | 根拠 |
|---|---:|---:|---:|---|
| paper_live | 1.0x | 0.40% | 0.60% | 通常運用 |
| m0 | 0.5x | 0.20% | 0.30% | R-05-03 / R-05-05 / R-39-02 |
| m1 | 0.75x | 0.30% | 0.45% | R-39-03 (デイトレ)、スイングは補完 |
| m2_plus | 1.0x | 0.40% | 0.60% | R-39-04 / R-39-05 |

## 重要な決定事項

- **M+1 スイング値は要件書未定義**: デイトレ倍率 0.75x を流用 (= 0.45%)。要件書に
  明示されたら修正する想定。
- **Stage 判定は手動**: `.env` の `OPERATIONAL_STAGE` を Fujiwara が更新。日数
  自動判定は別タスク (将来検討)。
- **F115 連携は当面なし**: F115 は引き続き PAPER_LIVE デフォルトで動く。Stage 3
  移行時に別タスクで F115 → F131 連携経路を作る。
- **enum は str 継承**: `OperationalStage("m0")` で文字列から作成可、JSON 化容易。
- **エラーで止めない**: 不正値はログ警告して PAPER_LIVE にフォールバック (運用
  中の事故防止)。
