# F130: 許容損失 準攻撃型実装 — 第 5 章正規ルール

## 概要

F115 TradeDecisionAgent の `_allowable_loss_per_trade()` 仮値 (equity × 0.5%) を、
**第 5 章 R-05-02/04/06 の準攻撃型正規ルール** に置き換え。
`risk/` パッケージ新規作成 (将来 F131 / F132 / F133 もここに集約)。

要件根拠:
- 第 5 章 R-05-01 (準攻撃型リスク管理)
- R-05-02 (デイトレ通常 0.4% / 強気 0.5% / 弱気 0.25-0.3%)
- R-05-04 (スイング通常 0.6% / 強気 0.8% 上限)
- R-05-06 (長期 1 銘柄最大 1.2%)
- R-05-07 (ロット計算式)

## 実装内容

### 主要モジュール (新規)

- `risk/__init__.py` (新規パッケージ): `calculate_allowable_loss` / `get_loss_rate` /
  `LOSS_RATE_TABLE` 等を export
- `risk/allowable_loss.py` (新規):
  - `LOSS_RATE_TABLE`: `(mode, regime) → rate` 辞書
  - `calculate_allowable_loss(equity, mode, regime, bear_severity=0.0)`: メイン関数
  - `get_loss_rate(mode, regime)`: レート単独取得 (UI 表示等用)
  - `DAYTRADE_BEAR_RATE_LOWER` / `_UPPER` 定数 (補間用)
- `agents/trade_decision.py` (修正):
  - `DEFAULT_ALLOWABLE_LOSS_RATIO` 定数値 0.005 → **0.004** (F130 連携反映)
  - `_allowable_loss_per_trade()` を `calculate_allowable_loss()` 呼び出しに置き換え
  - 関数内 import で循環回避

### 配置場所の最終決定

**`risk/` 新規パッケージ** を採用 (`agents/` ではない)。理由:
- F131 (M=0 半減)、F132 (日次・週次損失制御)、F133 (強制クローズ独立) も将来同パッケージに集約しやすい
- リスク管理ロジックは「エージェント」ではなく「制御層」として独立性を持たせる
- `from risk.allowable_loss import calculate_allowable_loss` で import パスが綺麗

### LOSS_RATE_TABLE (要件書根拠)

| mode | 通常 | 強気 | 弱気 | 出典 |
|---|---|---|---|---|
| daytrade | 0.4% | 0.5% | **0.25-0.3%** (補間) | R-05-02 |
| swing | 0.6% | 0.8% | 0.6% (要件書未定義、通常) | R-05-04 |
| longterm | 1.2% | 1.2% | 1.2% | R-05-06 (1 銘柄最大、分割買い前提) |

### デイトレ弱気の bear_severity 補間

R-05-02 は「弱気時 0.25-0.3%」と幅で表現されているため、`bear_severity: float`
(0.0-1.0) で連続補間:
- `severity=0.0` → 0.25% (最も保守側、F140 ゲート連携時のフォールバック)
- `severity=0.5` → 0.275% (中間)
- `severity=1.0` → 0.30% (連敗 3 回程度の想定)

将来 F110 Market Intelligence Agent が地合いから severity を算出する想定。
