# F279: 休日専業モード設計仕様書

## 概要

兼業前提で設計された FIRE 要件書 v3.3 に対し、平日休暇 / 在宅日に Mac mini 前で
張り付ける機会を活かす **休日専業モード**を新規追加する設計仕様書策定タスク。

スキャル + デイトレ戦略の解放、リスク許容度拡大、新規パターン定義、Pattern Store
Layer 階層改訂などを含む大規模設計。

## タスク詳細

設計議論記録は [[../03_design/operation_mode_holiday_intensive_draft_2026-05-04]] に集約済み (草案)。

### 確定方針 4 点 (Fujiwara 確認済 2026-05-04)

1. **本質**: 戦略の質的変化 (スキャル / デイトレ解放)、単純倍数延長ではない
2. **切替方法**: 手動 (Fujiwara がフラグ立て、当日有効、翌日リセット)
3. **アグレッシブ度**: 兼業比 回数 2.5x + 保有 3x + リスク 2x
4. **着手時期**: Stage 3 安定後 (M+1 以降)

### スコープ詳細 (草稿、F279 Phase 1 で確定)

- 5 モードスペクトラム ([休止][旅行][兼業][休日専業][フル専業 (将来)])
- 休日専業モードのパラメータ詳細表 (同時保有 8〜10 / エントリー 30〜40 /
  リスク 0.2-0.4% / score 緩和 -30% / etc)
- リスク管理ガード (連敗 3 / DD -2% / +5% で兼業強制降格)
- 切替運用ルール (タイミング 9:00 前 / 当日有効 / 翌日リセット)
- スキャル / デイトレ戦略の実装範囲
- Pattern Store Layer 設計判断 (mode 別 Active Set)
- モード切替永続化 (案 1 推奨: mode_state.json)
- Claude.ai 側からの懸念点 3 つ (再現性相性 / 心理負荷 / 生活習慣)
- 検証仮説 X / Y / Z

### Claude.ai 側からの懸念点 3 つ

1. **再現性優位戦略との相性**: スキャル系はエッジ小、ノイズ大。仮説 X/Y/Z で検証
2. **心理負荷**: 全通知 + 10 秒 tick + 8〜10 同時保有 = 高負荷、メンタル DD リスク
3. **生活習慣リスク**: 休日張り付き 6〜8 時間で本業以外の時間配分が不健全化

→ Phase 6 (1 ヶ月 Paper Live 実証) で確認、仮説 X が真なら F279 凍結。
