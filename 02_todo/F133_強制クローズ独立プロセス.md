   (`regime` 引数を受け取るのみで現状判定には未使用)。

## 重要な決定事項

- **配置**: `risk/time_guard.py` 独立関数 (F132 と同パターン)
- **「原則禁止」の解釈**: 全部禁止 (F110 未完成のため)。`regime` 引数は受け口
  のみ用意、将来拡張可能。
- **F115 内の判定順**: F133 (時刻、最上位) → F132 (損失) → F140 (執行)。
  時刻 cutoff は最も基本的な絶対制約なので最初。
- **既存テストの修正**: F115 既存 4 件 + F132 連携 3 件は `as_of_dt` が JST
  14:45 以降を使っていたため、`skip_time_guard=True` を追加してテスト意図を保持
  (F115/F132 単体ロジックの確認が目的だったため)。
- **自動クローズ実装は禁止**: Playwright や楽天 API 経由の自動発注クローズは
  v3.3 で撤回された方針。F133 でも書かない。
