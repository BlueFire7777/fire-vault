# F132: 日次・週次損失制御

## 概要

第 5 章 R-05-08/09/14 の累計損失制御を実装。F115 `decide_orders` 冒頭で
`check_loss_control` を呼び、停止状態なら全候補 rejected で即 return。

## スコープ

- 日次 -1.2% (新規停止) / -1.8% (当日終了) (R-05-08)
- 週次 -3% (守備モード) / -4% (大幅制限) / -5% (週内停止) (R-05-09)
- `LossControlState` dataclass で 5 フラグ + 損失率 + ラベル + 閾値リスト
- 損失率の分母は `paper_live_account.initial_capital` (なければ fallback)
- 「週」= 月曜〜金曜 (Asia/Tokyo)
- F115 `decide_orders` に `run_id` (Optional) と `skip_loss_control` 引数追加
  → 既存テスト 22 件は `run_id=None` で動くため非破壊

## 状態遷移マトリクス

| 損失率 | can_open_new | can_open_swing | must_close_today | week_disabled | defensive | label |
|---|---|---|---|---|---|---|
| 通常 | ✓ | ✓ | × | × | × | "通常" |
| 日次 ≤ -1.2% | × | ✓ | × | × | × | "新規停止" |
| 日次 ≤ -1.8% | × | ✓ | ✓ | × | × | "当日終了" |
| 週次 ≤ -3.0% | ✓ | ✓ | × | × | ✓ | "守備モード" |
| 週次 ≤ -4.0% | × | × | × | × | ✓ | "大幅制限" |
| 週次 ≤ -5.0% | × | × | × | ✓ | ✓ | "週内停止" |
| 日次 ≤ -1.2% かつ 週次 ≤ -3.0% | × | ✓ | × | × | ✓ | "新規停止 / 守備モード" |

※ 同一カテゴリ (日次 or 週次) 内ではより厳しい方を 1 つ採用。
※ 日次と週次は独立に判定し、両方ヒット時は label を `/` で連結。

## 重要な決定事項

- **paper_live_positions のカラム名は `exit_dt`** (仕様書の `closed_at` は誤り)。
  実 schema を確認してから SQL を書いた。
- **基準資産は `paper_live_account.initial_capital`** で代用 (簡易実装)。厳密には
  日次/週次始値を別管理するべきだが、F132 では不要 (将来 F260+ で改善)。
- **F115 引数追加は Optional**: `run_id=None` で既存テストを壊さない。Stage 3
  実弾運用時に PaperLive Runner / scheduler から `run_id` を渡せば自動で発動。
- **JST 変換**: `as_of_dt` を JST に変換してから日付決定 (UTC 23:00 = JST 翌日)。
  ただし SQL の DATE() は UTC 文字列の日付部分のみ抽出 (取引時間中は同一日)。
