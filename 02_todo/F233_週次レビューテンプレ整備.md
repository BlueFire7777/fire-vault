## タスク詳細

毎週日曜夜 (or 月曜朝) に Obsidian で記入する週次レビュー用テンプレ整備。
FIRE プロジェクトの進捗・成績・パイプライン健全性・次週計画を一画面で
俯瞰できることを目的とする。Stage 0〜3、M-3〜M+3 のどの月でも使える
1 枚テンプレに統合。

設計コンセプト:
- 時間軸: M-3 (Stage 0 開始 3 ヶ月前) 〜 M=0 (Stage 3 開始月) 〜 M+3
- 8 セクション構成 (ステージ位置 / 開発進捗 / 運用成績 / データ・パイプライン
  健全性 / リスク・ブロッカー / 学び・改善 / 来週計画 / メタ・ジャーナル)
- 記入対象セクションは Stage 別で目安提示 (Stage 0/1 は 3 をスキップ可、
  Stage 2 以降で 3 が主役化)
- 空欄・「該当なし」許容 (全項目必須にすると挫折する)

重要な決定事項:
- **配置**: `_templates/weekly_review/` 配下 (Vault ルートに `_` プレフィクス)。
  本来の成果物パスは `~/fire/docs/` だが、Obsidian で記入するため Vault
  内が運用上適切
- **Templater 不要**: Obsidian core Templates plugin の `{{date:YYYY-MM-DD}}`
  / `{{title}}` で動作。Templater 導入時は `<% tp.date.now() %>` 系へ
  置換可 (README 参照)
- **ファイル名規約**: `weekly_review_YYYY-MM-DD_WNN.md`
- **記入頻度**: 毎週日曜夜 21:00 推奨。月曜朝バックフィル可
- **初回運用日**: 2026-05-10 (日) 21:00 (Fujiwara 確定、events=0 事故の
  教訓を記憶が新しいうちに言語化する目的)
- **章番号付与なし**: テンプレは「番外」扱い、Chapter 38 と衝突しない

完了基準の引き上げ (2026-05-04、Fujiwara 指摘):
