# F233: M-3〜M+3 週次レビューテンプレ整備

## 概要

毎週日曜夜 (or 月曜朝) に Obsidian で記入する週次レビュー用テンプレ整備。
FIRE プロジェクトの進捗・成績・パイプライン健全性・次週計画を一画面で
俯瞰できることを目的とする。Stage 0〜3、M-3〜M+3 のどの月でも使える
1 枚テンプレに統合。

## 設計コンセプト

- **時間軸**: M-3 (Stage 0 開始 3 ヶ月前) 〜 M=0 (Stage 3 開始月) 〜 M+3
- **8 セクション構成** (1. ステージ位置 / 2. 開発進捗 / 3. 運用成績 /
  4. データ・パイプライン健全性 / 5. リスク・ブロッカー / 6. 学び・改善 /
  7. 来週の計画 / 8. メタ・ジャーナル)
- **記入対象セクションは Stage 別で目安提示** (Stage 0/1 は 3 をスキップ可、
  Stage 2 以降で 3 が主役化)
- **空欄・「該当なし」許容** (全項目必須にすると挫折する)

## 重要な決定事項

- **配置**: `_templates/weekly_review/` 配下 (Vault ルートに `_` プレフィクスで
  整理。本来の成果物パスは `~/fire/docs/` だが、Obsidian で記入するため
  Vault 内が運用上適切)
- **Templater 不要**: Obsidian core Templates plugin の `{{date:YYYY-MM-DD}}`
  / `{{title}}` で動作。Templater 導入時は `<% tp.date.now() %>` 系へ置換可
  (README 参照)
- **ファイル名規約**: `weekly_review_YYYY-MM-DD_WNN.md` (例: `weekly_review_2026-05-04_W18.md`)
- **記入頻度**: 毎週日曜夜 21:00 推奨。月曜朝バックフィル可
- **章番号付与なし**: テンプレは「番外」扱い。Chapter 38 (運用モード/休止/旅行モード)
  と衝突しない

### 関連スタブ (本タスクスコープ外、将来作成)

- `_templates/weekly_review/index.md` — 過去レビューのインデックス (4 件
  以上溜まったら起票)

## 設計上の意図

- **Stage 別の記入セクション目安をテンプレ冒頭に明示**: Stage 0/1 で
  運用成績欄が空白だらけになる挫折を防ぐ
- **TODO Excel との結合は ID + Wikilink**: テンプレ内では
  `[[F267_features_pipeline|F267]]` のような Wikilink で記入。02_todo/ の
  ノート未作成でも「リンク切れだが OK」運用 (将来作成すれば自動接続)
- **データ・パイプライン健全性は SQL 一発で出る項目に絞る**: README に
  推奨 SQL を列挙、コピペで現状値を取得できる
- **メタ・ジャーナルは任意**: Karpathy 式 log.md と棲み分け。週次は
  サマリ、log.md は時系列の生ログ
