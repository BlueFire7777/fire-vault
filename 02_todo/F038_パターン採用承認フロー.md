# F038: パターン採用承認フロー

## 概要

`patterns/approval.py` 新規作成。パターンのライフサイクル状態遷移 (採用/凍結/劣化/復活/再採用/フィルタ変更) を承認制で管理。自動反映禁止。

## 実装内容

### 主要モジュール

- `patterns/approval.py`: `ApprovalManager` + モジュールレベル公開 API
- `patterns/schema.sql`: `approval_requests` テーブル DDL (audit log、削除禁止)
- `agents/pattern_research.py`: `submit_decay/rehab/adopt_approval_request()` 追加

### 6 種類の承認イベント

- ADOPT (採用)
- FREEZE (凍結)
- DEATH_NOTE (劣化)
- REHAB (リハビリ)
- READOPT (再採用)
- FILTER_CHANGE (フィルタ変更)

### キーポイント

- audit log は追記専用 (削除禁止)
- 自動反映禁止 — Fujiwara の明示的承認が必要
