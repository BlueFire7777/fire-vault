# F044: 厳しめ約定モデル (C)

## 概要

`simulation/fill_models/strict.py` 追加。悲観シナリオ・板薄想定の約定判定。これで約定モデル 3 種 (ideal/realistic/strict) が完成。

## 実装内容

### 主要モジュール

- `simulation/fill_models/strict.py`: `StrictFillModel`
  - スリッページ: entry 0.20% / exit 0.30% (REALISTIC の 3〜4 倍、片道・不利方向)
  - 「越え」判定: TP/SL タッチだけでは不成立、本当に越えた場合のみ `filled=True`
    - long entry: `bar_low < target`
    - long exit: `bar_high > target`
  - 出来高チェック: `bar_volume < quantity × 10` で拒否 (不約定シミュレート)
  - 手数料・品質スコアは REALISTIC のヘルパー (`_calculate_rakuten_ichiban_commission` / `_compute_fill_quality`) を import で流用

### キーポイント

- factory に `_REGISTRY["strict"] = StrictFillModel` を 1 行で追加
- 「越え」 判定 = touched では不十分という設計が悲観性の本質
- F046 で 3 者比較 (ideal/realistic/strict) が可能になった
