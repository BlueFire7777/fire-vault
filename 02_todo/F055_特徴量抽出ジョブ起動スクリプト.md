# F055: 特徴量抽出ジョブ起動スクリプト (F024 + F026 ミニマル版)

## 概要

F100 で流入した market_prices_daily から features テーブルを埋めるバッチジョブ。
F024 (テクニカル) + F026 (時間帯) をミニマル版として実装し、F031 類似局面検索の
入力源を確保。F021/F022/F023/F025 はデータソース未整備で「skipped」明示。

要件根拠:
- 第 36 章 R-36-01〜R-36-10 (初期特徴量セット)
- 第 14 章 (データソース優先順位)

## 実装内容

### 主要モジュール

- `scripts/jobs/__init__.py` (新規パッケージ)
- `scripts/jobs/extract_features.py` (新規本体)
  - `COLLECTOR_STATUS`: 各 collector の ready/skipped 定義 (将来拡張点)
  - `fetch_price_data(code, date_from, date_to, db_path)`: market_prices_daily 取得
  - `run_technical_collector(code, date, price_history, db_path)`:
    F024 TechnicalFeatureCollector へ TechnicalData を組み立てて呼び出し
  - `run_time_collector(code, date, db_path)`: F026 TimeFeatureCollector
  - `extract_features(code, date, collectors, db_path)`: メイン関数 (status 辞書返却)
  - `_to_feature_dt(date)`: features.dt 規約 = `"{date}T09:00:00+09:00"` (JST 寄付)
  - CLI: `--code / --date / --date-from / --date-to / --collectors`

### COLLECTOR_STATUS

| collector | ready | 理由 |
|---|---|---|
| material  | ❌ | F101 (TDnet) 未着手 |
| regime    | ❌ | F104 (指数取得) 未着手 |
| liquidity | ❌ | 板データ取得タスク未着手 |
| **technical** | ✅ | F100 流入データで動作 |
| execution | ❌ | 楽天証券データ取得未着手 |
| **time**  | ✅ | datetime のみで動作 (入力データ不要) |

### キーポイント

- **F031 のための入力**: features テーブルが埋まることで F031 SimilarityEngine が
  実データで稼働可能になる (F056 で動作保証済)
- **データ 0 件で空完走**: 履歴 0 → technical は count=0、例外なし
- **休日ハンドリング**: market_prices_daily に該当日が無い (祝日等) → count=0 で完走
- **features.dt 規約**: `"{date}T09:00:00+09:00"` = 当該営業日の寄付き JST 時刻
- **status 3 種**: `ok` / `skipped` (data source 未整備) / `error` (例外)

### F021-F026 のコード読解で気づいた点 (実 API vs 指示書)

| 観点 | 指示書の想定 | 実 API |
|---|---|---|
| `Bar` フィールド | `date` | **`dt`** |
| `TechnicalFeatureCollector.collect` | `collect(symbol, dt, today_bar, history)` | **`collect(symbol, dt, data: TechnicalData = None, **context)`** (TechnicalData オブジェクトを 1 つ渡す) |
| `TimeFeatureCollector.collect` | `collect(symbol, dt, eval_dt)` | **`collect(symbol, dt, material_published_at=None, forced_close=...)`** (eval_dt は dt から内部パース) |
| 戻り値 | int (count) | **dict** `{"collected_count": N, "features": dict, ...}` |

→ extract_features.py 内で実 API に合わせて TechnicalData を組み立てている。
