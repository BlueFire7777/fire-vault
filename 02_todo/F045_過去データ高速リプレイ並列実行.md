# F045: 過去データ高速リプレイ並列実行

## 概要

`simulation/replay/parallel.py` 新規作成。F041 `ReplayRunner` をラップして `ProcessPoolExecutor` で並列実行する `ParallelReplayRunner`。F041 は非改変。

## 実装内容

### 主要モジュール

- `simulation/replay/parallel.py`: `ParallelReplayRunner`
  - パターン並列メイン + 期間分割並列オプション (`--split-by-period`)
  - `parent_run_id` でサブジョブをグループ化 (`PRP-` prefix)
  - `_run_single_subjob` はトップレベル関数 (ProcessPoolExecutor の pickle 要件)
  - デフォルト `max_workers = os.cpu_count() // 2` (他タスク圧迫回避、最低 1 にクランプ)
- `scripts/setup/migrate_parallel_replay.py`: 既存 `replay_runs` / `replay_results` に `parent_run_id` カラム追加 (後方互換)
- `simulation/replay/cli.py`: `--parallel` / `--split-by-period` / `--workers` フラグ追加
- F046 で追加: `fill_models` オプション (複数モデル並列) — 互換性保証のため未指定時は `["ideal"]` にフォールバック

### キーポイント

- F041 ReplayRunner は非改変 (既存 20 PASS 保護)
- 親 run 用 ID prefix `PRP-` で子 run の `RP-` と区別
- `models_to_run` パラメータで F046 の 3 モデル並列を支える

## テスト
