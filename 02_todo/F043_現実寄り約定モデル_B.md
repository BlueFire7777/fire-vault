# F043: 現実寄り約定モデル (B)

## 概要

`simulation/fill_models/realistic.py` 追加。固定率スリッページ + 楽天証券いちばんコース手数料 + 約定不成立判定 (TP/SL タッチ) を実装。

## 実装内容

### 主要モジュール

- `simulation/fill_models/realistic.py`: `RealisticFillModel`
  - スリッページ: entry 0.05% / exit 0.10% (片道、不利方向)
  - 手数料: 100 万円まで 0 円 / 100-200 万 2200 円 / 200-300 万 3300 円 / 超過 1100 円/100 万円
  - 約定不成立判定: target_price が `bar_low〜bar_high` 範囲外なら `filled=False` (TP/SL タッチ判定)
  - `fill_quality_score`: スリッページ・手数料減点で 0.0〜1.0 にクランプ

### キーポイント

- F042 の factory 基盤を `_REGISTRY["realistic"] = RealisticFillModel` の 1 行で拡張
- ヘルパー関数 (`_calculate_rakuten_ichiban_commission` / `_compute_fill_quality`) は F044 から import で流用される設計
