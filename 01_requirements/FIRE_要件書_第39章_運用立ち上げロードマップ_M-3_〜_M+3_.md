# 第39章: 運用立ち上げロードマップ(v3.3: M-3 〜 M+3)

## v3.3 での変更

v3.1/v3.2 のロードマップはそのまま有効だが、Stage 3 の内容が「Semi Auto(承認制の自動発注)」から「**Live Advisory(LINE通知 → 人間発注)**」に変わる。Stage 4 (Full Auto) は将来オプションとして位置づけを下げる。

## ロードマップ(v3.3)

実資金を投入して Live Advisory 運用を開始する月を「M=0(運用初月)」と定義し、以下のスケジュールで段階的に立ち上げる。

| 期間 | Stage | 実弾 | デイトレリスク | 主要アクション |
|---|---|---|---|---|
| M-3 〜 M-1 | Stage 0/1/2 | 未投入 | — | Mac mini常時稼働による過去データ高速リプレイ検証とリアルタイム Paper Live を並列実行し、パターンを爆速蓄積する |
| **M=0**(運用初月) | **Stage 3 (Live Advisory)** | 投入 | 0.2%(半減) | **LINE通知を見て Fujiwara さんが手動発注**。執行実測データ収集のため、デイトレリスクを半減でスタート。初月からの黒字化を目指す |
| M+1 | Stage 3 | 投入 | 0.3% | 執行乖離が想定内であることを確認し、リスク枠を拡大。**M+2 以降に向けて Gmail API での約定自動取込を準備** |
| M+2 〜 M+? | Stage 3 | 投入 | 0.4%(要件書通り) | フルリスク運用。**Gmail API で約定自動取込開始**。Pattern Library の浄化と期待値の安定化 |
| M+3 以降 | Stage 3 継続 | 投入 | 0.4% | **Live Advisory が FIRE の完成形。Stage 4 (Full Auto) は必要性がでたら検討(オプション)** |

## v3.1 からの重要な変更点

| 項目 | v3.1 | v3.3 |
|---|---|---|
| M+3 の目標 | Stage 4 (Full Auto) に移行、デイトレSランクのみFull Auto | **Stage 3 (Live Advisory) 継続**、Stage 4 は将来検討 |
| 発注手段 | 楽天証券 API 経由で自動発注 | **LINE通知 → 人間発注** |
| 強制クローズ | システム自動 | **LINE 緊急アラート → 人間対応** |

## M=0 移行前のチェックリスト(v3.3)

Paper Live → Live Advisory 昇格時の必須条件(第12章・第32章参照):

- [ ] Paper Live で 20営業日以上 or 50トレード以上のサンプル蓄積
- [ ] 主戦略の期待値がプラス
- [ ] 最大ドローダウンが想定内
- [ ] Simulation Optimistic Bias Score が許容内
- [ ] LINE 通知基盤が全5部屋で正常動作(エントリー/執行損益/レポート/システム警告/緊急ポジション整理)
- [ ] 緊急ポジション整理アラート(14:45/14:55/15:05/15:10/15:15)が動作確認済み
- [ ] Fujiwara さんが LINE 通知を本業中にも確実に確認できる体制を構築済み
- [ ] Evaluation Agent が昇格提案書を作成済み
- [ ] Fujiwara さんが最終承認
