# 第35章: Mac mini 常駐基盤の時間帯別分担

## 「24時間稼働」の正しい理解

Mac miniは24時間起動しているが、**24時間ずっと高負荷で動くわけではない**。常駐基盤(FIRE Runtime)が時間帯に応じて必要な処理を起動する構成。

## 常駐コンポーネント

| 常駐コンポーネント | 役割 |
|---|---|
| スケジューラ | 時間帯ごとに処理を起動・停止 |
| 市場監視ワーカー | 取引時間中のリアルタイム監視 |
| Simulation / Paper Liveワーカー | 取引時間中の仮想売買 |
| Pattern Researchワーカー | 夜間の類似局面研究・パターン更新 |
| Notificationワーカー | LINE通知・朝夕レポート送信 |
| Dashboard Backend | 常時稼働(軽量) |

## シミュレーション種別ごとの稼働タイミング

| 種別 | 稼働タイミング |
|---|---|
| Backtest | 常時不要。夜間バッチまたは手動実行 |
| Replay Simulation | 常時不要。指定日を再生する時だけ起動 |
| Paper Live | 取引時間中は常時稼働(08:00〜15:30) |

## 時間帯別の動作内容

| 時間帯 | 動作内容 |
|---|---|
| 朝(〜09:00) | ニュース取得 / 開示取得 / 候補抽出 / 朝レポート生成・送信 |
| 場中(09:00〜15:30) | Paper Live simulation常時稼働 / 条件到達監視 / 仮想エントリー / 仮想利確・損切 / 仮想強制クローズ |
| 引け後(15:30〜) | 当日集計 / Dashboard更新 / Evaluation Agent用データ生成 / 引け後レポート送信 |
| 夜 | Pattern Research(類似局面更新)/ DEATH NOTE候補抽出 / Backtest・Replay(必要時)/ 軽い分析バッチ |

## 運用イメージの一文要約

Mac miniの常駐エージェント群がシミュレーションを回すイメージでOK。ただし、Paper Liveは相場時間中心・Backtest/Replayは必要時・夜は研究バッチ中心、という**時間帯別の分担**にするのがベスト。
