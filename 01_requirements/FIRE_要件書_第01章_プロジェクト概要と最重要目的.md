# 第01章: プロジェクト概要と最重要目的

## プロダクト名
**FIRE**

## 最重要目的
ユーザーが本業中でも相場に張り付かず、**LINE通知を見た瞬間に発注設定が完結**し、再現性高く利益を最大化すること。個別トレードの天井を取り切ることより、兼業前提での**月次・年次利益の最大化**を優先する。

## UIテーマ
レトロな取引端末風(CRTインスパイア、警告灯・点火ランプ・発光アクセントとしての炎モチーフ)。

## 資産目標
総資産**3,000万円を2年で達成**する。目標未達時も無謀なリスク拡大は行わず、売買の質・頻度・配分の改善を優先する。

## 乖離率別の対応方針

| 乖離率 | 判定 | 対応方針 |
|---|---|---|
| +5%以上 | 先行 | 無理に攻めない |
| -5%以内 | 計画線上 | 現状維持 |
| -5%〜-10% | 軽度の遅れ | 改善検討 |
| -10%〜-20% | 要改善 | 戦略見直し |
| -20%以下 | 戦略再設計 | 配分・ルール・頻度を再設計 |

## v3.3 の重大な方針転換(v3.1/v3.2 からの再定義)

### 変更の背景

v3.1/v3.2 は「完全自動発注」を前提に設計されていたが、2026年時点の日本株環境では以下の構造的障壁がある:

- 楽天証券公式 REST API は存在しない
- マーケットスピード II RSS は Windows + Excel 必須(Mac 非対応)
- 2025年6月から全ログイン経路で 2FA 必須化
- Computer Use によるブラウザ自動化は、執行速度・誤発注・UI変更追従・規約グレーの4重リスク

これらを踏まえ、**完全自動化は「エッジ確立後のオプション」**とし、**主軸は半自動(LINE通知 + 人間発注)に転換**する。

### v3.3 の3大変更

1. **OpenClaw を採用**(オーケストレーション層として必須)- v3.2 方針の継続
2. **Computer Use による自動発注は廃止** - 発注は人間(Fujiwara さん)が LINE通知を見て実行
3. **独立 Cron + Playwright による強制クローズは廃止** - 強制クローズも LINE 緊急アラートで人間対応

### 変更しない本質(FIRE の核心)

- ✅ Pattern Store / Feature Store / 再現性エンジンは変わらず最優先
- ✅ OpenClaw による 24時間常駐は継続
- ✅ Mac mini M4 Pro 前提
- ✅ 崩してはならない前提8点(第15章)も本質は同じ

### v3.2 で採用した変更のうち、v3.3 で撤回するもの

- ❌ Computer Use による楽天証券のブラウザ直接操作
- ❌ 注文確認画面のスクリーンショット → LLM セルフレビュー
- ❌ 独立 Cron + Playwright による強制クローズの自動化

### v3.3 で採用・強化するもの

- ✅ OpenClaw オーケストレーション層
- ✅ LINE 通知による発注指示(注文完成形)
- ✅ **LINE 緊急ポジション整理部屋**(14:45/14:55/15:05/15:10/15:15 の段階アラート)
- ✅ 寄り前指値予約運用(会議中対応困難時の機会損失対策)
- ✅ 楽天証券約定通知メール → Gmail API 自動取込(M+2 以降)
- ✅ Google Calendar 連携 or 朝一言送信による当日スケジュール共有
- ✅ OpenClaw 障害時の縮退運転設計(完全停止機能 vs 稼働継続機能の分離)

## v3.2 までの主な追加(既にマージ済み)

### v3.0
- 運用立ち上げロードマップ(M-3 〜 M+3)
- 過去データ高速リプレイとリアルタイム Paper Live の並列実行
- Full Auto 移行時のシグナルベース重複防御(5層)
- 開発着手順の最適化(Pattern Store 最優先)

### v3.1
- 主戦略(材料初動 80%)と準戦略(主役テーマ連動値嵩株 20% / M+2までPaper Live並走)の優先順位明確化
- ハイブリッド執行方針(Sランクはブレイク狙い、通常は指値主体)
- Paper Live の厳しめ約定モデル
- ダッシュボード上部への「昇格ブロッカー」常時表示

### v3.2
- OpenClaw オーケストレーション層採用
- Computer Use による発注(v3.3 で撤回)
- 独立 Cron による強制クローズ(v3.3 で撤回)

### v3.3(今回)
- 発注の半自動化(人間中心)
- 強制クローズの LINE 緊急アラート化
- OpenClaw は継続採用(オーケストレーション層の役割に限定)
