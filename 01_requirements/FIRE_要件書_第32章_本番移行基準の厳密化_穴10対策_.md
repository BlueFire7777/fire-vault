# 第32章: 本番移行基準の厳密化(穴10対策)

## 穴10の本質

Paper Liveを入れるのはよいが、**「どこまで行けばsemi_autoに上げていいか」が曖昧**だと危ない。明確な昇格条件が必要。

## Paper Live → Semi Auto 昇格条件(完全版)

以下の**全条件**を満たすことが必須。

| 条件 | 基準 |
|---|---|
| サンプル数 | 20営業日以上 または 50トレード以上(※Mac miniの常時稼働を活かし、過去データの高速リプレイ検証とリアルタイムPaper Liveを並列実行することで、カレンダー日数を待たずに条件クリア可能とする。過去データで20営業日以上かつ50トレード以上の基準を満たせば、件数と期間の昇格条件をクリアしたものとみなす) |
| 主戦略の期待値 | 期待値がプラス(勝率だけでなく平均利益・平均損失も含めてプラス) |
| 最大ドローダウン | 想定上限以内(Paper上で総資産比 -5% を大きく超えない) |
| Simulation Optimistic Bias Score | 許容上限以内(理想約定と現実約定の乖離が小さいこと) |
| 強制クローズ成功率 | 設定した強制クローズルールが正常に機能していること |
| 停止機能の動作確認 | 日次損失上限・週次損失上限・強制クローズが全て正常動作 |
| Evaluation Agentの確認 | 明確な改善提案が残っていないこと(または対処済みであること) |

## Semi Auto → Full Auto 昇格条件

| 条件 | 基準 |
|---|---|
| Semi Autoでの実績 | 20営業日以上 または 50トレード以上 |
| 期待値 | Semi Auto期間中もプラスを維持 |
| 最大ドローダウン | 想定上限以内 |
| 承認率 | Semi Autoでの承認率が安定している(ほぼ全承認 or 却下理由が明確) |
| システム安定性 | 重大障害・誤発注・データ欠損なし |

## 昇格の原則

- **Stage飛ばし禁止**: Stage 0 → 1 → 2 → 3 → 4 の順序を必ず守る
- **昇格はユーザーが承認**: 自動昇格は禁止
- **降格は即時**: 重大障害・連続損失・停止機能異常が発生したら即座に前のStageに戻す

- **R-32-02** Semi→Full昇格条件
- **R-32-03** 昇格原則(Stage飛ばし禁止・ユーザー承認・降格即時)
