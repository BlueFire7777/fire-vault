# 第06章: LINE通知設計(v3.3: 半自動主軸の注文完成形)

## v3.3 での重要性

v3.3 では発注が半自動(人間中心)となるため、**LINE 通知は FIRE と Fujiwara さんの唯一の発注指示経路**となる。通知品質が運用品質を直結して決める。

## 通知部屋の分け方(v3.3: 5部屋体制)

| # | 部屋名 | 用途 |
|---|---|---|
| 1 | **エントリー通知** | 新規候補、条件成立、発注指示(注文完成形) |
| 2 | **執行・損益通知** | 利確指示、損切指示、約定確認、当日損益更新 |
| 3 | **レポート通知** | 朝レポートPDF、8:40候補一覧、引け後レビューPDF |
| 4 | **システム警告** | エラー、データ取得失敗、FIRE停止、OpenClaw異常 |
| 5 | **🚨 緊急ポジション整理部屋**(v3.3新設) | 14:45〜15:15 の段階的クローズアラート |

## 通知1通に含める必須項目(発注指示時)

銘柄名 / 売買区分(買/売・新規/返済) / 行動区分(即発注・条件確認後・監視のみ・取消・修正・半分利確・全利確・損切実行・見送り) / 新規価格 / 株数 / 逆指値 / 利確価格(分割設計) / 損切価格 / 有効期限 / 無効条件 / 一言戦略

## 通知例(200株建ての場合)

```
【エントリー通知】
銘柄: ○○
行動: 即発注
新規: 3,420 で 200株(信用買建)
逆指値: 3,360
利確1: 3,520 で 100株
利確2: 3,580 で 100株
有効期限: 10:15 まで
無効条件: 3,400 を明確に割れたら見送り
戦略: 前場高値ブレイク初動狙い
```

※ **100株建ての場合は原則として利確1回+逆指値管理**

## 🚨 緊急ポジション整理部屋(v3.3 新設)

### 目的

Fujiwara さんが本業中でも 15:10 までに確実に全信用建玉をクローズできるよう、段階的にアラートを送る。v3.2 の「独立 Cron + Playwright による自動クローズ」を置き換える運用である。

### アラート時刻と内容

| 時刻 | 区分 | 通知内容 |
|---|---|---|
| **14:45** | 第一次通知 | 「建玉残 X 件、15:10までにクローズしてください」 |
| **14:55** | 第二次通知 | 「建玉残 X 件、あと15分」 |
| **15:05** | 第三次通知(最重要) | 「⚠️ 残り5分、即クローズを」 |
| **15:10** | 期限到達通知 | 「強制クローズ期限到達。建玉残 X 件」 |
| **15:15** | 最終確認 | 「15:10時点でクローズ完了を確認してください」 |

### 運用フロー

1. 14:45 通知をスマホで確認(本業中でも即確認可能な時刻設定)
2. 会議中・離席中でも、15:10までの約25分でトイレ休憩やスキマ時間に iSPEED/楽天Web から成行クローズ
3. クローズ完了を Fujiwara さんが LINE に「クローズ済」と返信、または Dashboard で「完了」チェック
4. FIRE がログに反映

### 運用ルール補強(建玉持越し防止)

アラート自動化に頼らないため、運用ルール自体で持越しリスクを下げる:

- **原則、前場(9:00〜11:30)のみ新規エントリー**
- **後場(12:30〜15:30)は既存建玉の管理のみ、新規エントリー禁止**
- **11:30 前場引け時点で建玉ゼロを目標**(デイトレ短期決着型)
- **Fujiwara さんのスケジュール上 15:00〜15:10 が対応困難な日は、前場引けで全クローズ**を優先

## 約定後の報告フロー

### 初期(M=0 〜 M+1): LINE 手動報告

Fujiwara さんが発注後、約定したら LINE 通知部屋(執行・損益通知)に以下を返信:

```
【約定報告】
シグナル ID: XXXX
約定価格: 3,425
約定株数: 200株
約定時刻: 9:23
部分約定: なし
```

FIRE がこれを解釈し、建玉データベースに記録。

### M+2 以降: Gmail API 自動取込

楽天証券の約定通知メール設定を有効化し、Gmail API で FIRE が自動取込。

**楽天証券側の設定**:
- Web サイト → 設定・変更 → お知らせメール設定 → 注文約定通知を ON
- メール内容: 約定日時、銘柄コード、売買区分、新規/返済、約定価格、約定数量、現物/信用区分

**FIRE 側の処理**:
- Gmail API で約定通知メールを受信
- メール内容をパース、建玉データベースに自動記録
- FIRE 想定と実約定の差分(シグナル価格と約定価格の乖離)を検知・記録(執行品質スコアに反映)

## レポート出力(v3.1 継承)

### 朝レポート (8:30 LINE送信・PDF)
NotebookLM風の視覚的生成AIレポート。3行要約 / 市況 / 日本株セクター資金動向 / **ニュースのトレード洞察変換** / **高確率トレード機会 5件**(セットアップ、提案エントリー、利益目標、損切、想定RR、根拠) / ポートフォリオ偏り点検。

### 売買シグナルの強化
主要サポート/レジスタンスゾーン、トレンドライン、移動平均、モメンタム指標を特定し、「買い・ホールド・売り」を明確に示す。**朝レポートでは詳細な推論、LINE通知では結論のみ**。

### 引け後レビュー (16:00 LINE送信・PDF)
売買履歴、実現損益、良し悪し、改善点、再発防止ルール、目標進捗との乖離、ポートフォリオ変化。

## 通知疲れ防止

日に何度も通知が来ると本業に支障が出るため、以下のルール:

- **緊急・重要のみ即時通知**(エントリー指示、約定確認要、緊急クローズアラート)
- **情報通知は朝レポート / 引け後レポートに集約**(市況コメント、個別材料の詳細、軽微なセクター変動など)
- **通知頻度の目標**: 日次トレード数 0〜3件 × (指示 + 約定確認 + 利確/損切) = 日中最大 10件程度
- **音設定**: エントリー通知と緊急ポジション整理部屋だけ音ON、それ以外は音OFF推奨(LINE 側設定)
