# 第20章: FIREの勝率優位性の根拠

FIREは「確実に勝てる」システムにはならない。ただし、**勝率や期待値が高くなりやすい局面だけを狙う設計**にはできる。FIREの優位性は未来予知ではなく、以下の**4つ**に置く。

## 1. 情報優位

| 情報源 | 役割 | なぜ効くか |
|---|---|---|
| TradingView | チャート・相対強弱・主役テーマ把握 | 入る位置が機械化できる。無効条件が決めやすい。RRを計算しやすい |
| 楽天証券 マーケットスピードII RSS | リアルタイム価格取得・発注機能。余力・建玉・注文状態・約定履歴 | 現実に買える株数でしか発注しない。建てすぎを防ぐ。紙の上だけ勝つのを防げる |
| TDnet / IR API | 適時開示・決算短信 | 単なるテクニカルより資金が集まる理由がある。一過性ノイズと本物の再評価を分けやすい |
| ニュースAPI | 市況・個別材料 | 材料の鮮度と質を判定できる |
| LINE Messaging API | 通知・承認・停止コマンドのUI | 実行結果が見える。停止ができる。承認制にできる。異常を見逃しにくい |

FIREは、ニュース・開示・チャート・価格・約定環境を別々に見ず、**1つの戦略判断に統合する**。これがまず強い。

## 2. 監視優位

人間が弱いのは「同じ条件を漏れなく、何時間も監視すること」。FIREはここで勝ちやすい。

**監視対象**: 好決算・上方修正・自社株買い・増配 / 主役テーマへの資金流入 / 値嵩株の前場初動 / エントリー価格到達 / 利確価格到達 / 損切価格到達 / 時間切れ / システム異常

## 3. 執行優位

FIREは候補が良いだけでなく、**執行ルールが固定されている**から優位になりやすい。

**勝率が上がりやすい執行条件**: デイトレ候補を1〜3銘柄に絞る / 損切を最初から固定する / 逆指値を必須にする / 14:45以降の新規を止める / 15:10までに信用を閉じる / 地合いが悪ければ自動停止する

これは「**当てる力より負け方を改善する力**」に近い。

## 4. 停止優位

多くの裁量トレードが壊れるのは「良いシグナルがないから」ではなく「**止まれないから**」。

**止める条件**: 日次損失上限 / 週間損失上限 / 楽天接続異常 / 市場データ異常 / 重複注文 / 板異常 / 強制クローズ失敗

FIREは「勝つシステム」というより、**「壊れにくいシステム」**として勝率や期待値を上げる。

## FIREの第一エッジ(一文要約)

FIRE は、TradingView で可視化される日本株のチャート・相対強弱・地合い、ニュースAPIとTDnet / IR由来の高品質材料、楽天証券連携で取得する余力・建玉・注文状態を統合し、好決算・上方修正・還元強化の初動、または主役テーマ連動の値嵩株前場初動だけを対象に、機械的な損切・利確・停止ルールのもとで自動執行することで、人間の監視漏れ・判断ブレ・停止遅れを減らし、勝率と期待値の改善を狙う。

## FIREがエッジにしないもの(明文化)

- 有名トレーダーのコピトレを主戦略にすること
- SNSの煽りを主材料にすること
- 板の微差を抜く超高頻度スキャル
- 純粋な裁定戦略
- 未来予測だけに依存すること

## 勝てそうと言える条件(確認チェックリスト)

| 確認項目 | 内容 |
|---|---|
| 主戦略が明確 | 好決算・上方修正・還元強化の初動、主役テーマ値嵩株の前場初動 |
| 使う情報源が明確 | TradingView・楽天証券連携・市場データAPI・ニュースAPI・TDnet/IR・LINE API |
| 情報の役割分担が明確 | TradingView=分析、楽天証券=口座・注文の正本、LINE=通知と停止、ニュース/開示=材料判断 |
| 執行ルールが明確 | 指値中心・損切必須・時間制限あり・強制クローズあり |
| やらないことが明確 | コピトレ主戦略化・純スキャル・超高頻度・曖昧な裁量 |
| 検証モードがある | backtest → replay → paper live → semi auto → full auto |

## 本章の要件ID

- **R-20-01** 情報優位 5情報源統合
- **R-20-02** 監視優位 漏れない監視対象
- **R-20-03** 執行優位 固定ルール
- **R-20-04** 停止優位 止まれる
- **R-20-05** エッジにしないもの
- **R-20-06** 勝てそうと言える条件チェックリスト7点
