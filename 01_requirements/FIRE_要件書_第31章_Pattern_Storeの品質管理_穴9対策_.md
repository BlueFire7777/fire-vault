# 第31章: Pattern Storeの品質管理(穴9対策)

## 穴9の本質

勝ちパターンを溜めるのはよいが、**重複パターン・似すぎたパターン・名前だけ違うパターン**が増えると使い物にならなくなる。

## 品質管理の必須要素

| 要素 | 内容 |
|---|---|
| パターン命名規則 | `材料種別_時間帯_地合い_バージョン` の形式で統一 |
| 類似パターン統合ルール | 特徴量の重複度が閾値以上のパターンは統合候補とする |
| pattern_id 管理 | 一意のIDで管理。変更・廃止の追跡に使う |
| バージョン管理 | パターンの条件変更時はバージョンを上げる(旧バージョンの成績は保持) |
| 廃止履歴 | DEATH NOTEに送ったパターンの廃止理由・日付・当時の成績を保存 |

## パターン数の上限(再掲)

| 区分 | 上限 |
|---|---|
| Primary Active Patterns | 2〜4(初期) |
| Candidate Patterns | 5〜10 |
| DEATH NOTE / REHAB | 制限なし(ただし定期的に棚卸し) |

## 本章の要件ID

- **R-31-01** パターン命名規則
- **R-31-02** 類似パターン統合ルール

- **R-31-04** バージョン管理
- **R-31-05** 廃止履歴
- **R-31-06** パターン数上限
