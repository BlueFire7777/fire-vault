# 第13章: Evaluation Agent と Dashboard

## Evaluation Agent の目的と位置づけ

FIREは候補抽出・執行・停止・レビューと複数のエージェントが連携するため、「どこが悪かったのか」が人間には分かりづらくなる。そこで、各エージェントの判断と結果を採点し、改善提案を行う**Evaluation Agent(自動反省エージェント)**を導入する。

Evaluation Agentは、生成AIを用いてユーザーが短時間で理解できる評価提案書を作成し、ユーザーへ提案することを目的とする。**ルール変更の自動反映は行わず、改善提案の採用可否は必ずユーザー承認制**とする。

## Evaluation Agent の評価対象

| 評価カテゴリ | 主な確認内容 |
|---|---|
| 候補抽出評価 | 朝の5候補は妥当だったか、1〜3銘柄への絞り込みは適切だったか、見送り銘柄の方が良かったか |
| 執行評価 | エントリータイミング、指値の適切さ、利確の早遅、損切の妥当性、強制クローズの適切さ |
| リスク管理評価 | ロットの適切さ、地合いに対して攻めすぎていなかったか、停止判断の遅れ、テーマ集中・セクター偏りの過大 |
| システム評価 | 監視遅延、データ欠損・接続障害、発注失敗・重複注文、自動停止の正常動作 |
| エッジ別評価 | 好決算/上方修正/還元強化初動の成績、主役テーマ連動値嵩株前場初動の成績、どのエッジが機能したか |

## 評価提案書の出力方針

- **NotebookLM風**に視覚的に分かりやすいレポートとする
- 冒頭で重要論点を**3行で要約**
- 良かった点・悪かった点・改善提案を明確に分ける
- 専門的すぎる説明より、実務で判断しやすい表現を優先
- 変更候補は「何をどう変えるべきか」を具体的に記載
- **自動反映は行わない**

## 承認制ルール

Evaluation Agentは以下の変更を自動反映してはならない。採用はユーザー承認後にのみ設定へ反映する。

**承認対象**: スコア配点変更 / 候補抽出条件変更 / 利確・損切条件変更 / ロット調整ルール変更 / 停止条件変更 / エッジ定義変更 / ダッシュボード閾値変更

## 評価の時間軸

| 時間軸 | 内容 |
|---|---|
| 日次評価 | 当日の良かった点・悪かった点、翌日にすぐ活かせる軽微な改善提案 |
| 週次評価 | エッジ別成績の比較、候補抽出精度の傾向、地合い別の成績差、ルール修正候補 |
| 月次評価 | 勝ち筋の有効性評価、ロット配分の妥当性、自動停止ルールの適切性、大きな設計変更候補 |

## 後知恵バイアス対策

Evaluation Agentは後知恵バイアスを避けるため、評価時には**当時利用可能だった情報のみ**を使って評価することを原則とする。「結果的に上がったから採用すべきだった」という単純評価を避け、その時点で合理的だったかどうかを重視する。

## Dashboard の方針

FIREの運用状態、損益、候補精度、執行品質、リスク管理、システム状態、Evaluation Agentの改善提案を一元的に確認できるDashboardを導入する。Dashboardの目的は単なる損益表示ではなく、**FIREが今どの程度うまく機能しているかを短時間で把握できる運用コンソール**を実現することである。

### 基本思想
LP的な装飾よりもトレーディング運用コンソールとしての実用性を優先する。

### 主要機能(8カテゴリ)

| カテゴリ | 表示内容 |
|---|---|
| 運用状況 | 稼働中/停止中、自動運用モード、本日の実行候補数、建玉数、新規停止、異常状態 |
| 損益 | 日次・週次・月次・累積損益、実現/含み損益、最大ドローダウン |
| 目標進捗 | 現在総資産、理想総資産、乖離額・乖離率、日/週/月の必要進捗 |
| 候補精度 | 朝の高確率機会5件の成績、実行候補1〜3件の成績、見送り候補の結果、候補抽出精度の推移 |
| 執行品質 | 指値約定率、利確到達率、損切率、強制クローズ率、想定価格と実約定の乖離、注文失敗率 |
| リスク管理 | 日次/週間停止発動回数、自動停止理由、同一テーマ集中度、セクター偏り、ロット適正度 |
| システム状態 | 楽天証券・市場データ・TradingView MCP・LINE・ニュースAPI・開示APIの接続状態、発注成功率、監視遅延 |
| Evaluation Agent | 今日/今週/今月の改善提案、承認待ち/採用済み/却下済み履歴 |

### 画面構成(7ブロック)

A: ヘッダーサマリー / B: 今日の運用状況 / C: エッジ別成績 / D: 候補抽出と執行の品質 / E: リスクとポートフォリオ / F: Evaluation Agent提案 / G: システム状態

### 状態表示の優先度
**危険/停止中 > 異常あり > 承認待ち > 稼働中 > 参考情報**

### 初期実装範囲
**初期必須**: 今日の損益、日次/週次/月次損益、目標乖離率、今日のデイトレ候補、実行候補の勝率、エッジ別成績、自動停止回数、楽天接続状態、LINE送信状態、Evaluation Agentの提案一覧、承認待ち提案一覧
