# 第41章: セクター/銘柄特有性対応(穴12対策)  
  
## 穴12の本質  
  
FIREのパターン命名規則(R-31-01)は `material-regime-price_action-time-liquidity` の5セグメントだが、これだけでは**セクターや特定銘柄群でしか機能しないパターン**を表現できない。  
  
例:  
- 半導体株の米SOX指数連動寄付ギャップ  
- トヨタの為替連動(USD/JPY 突発変動時の寄付)  
- 海運株のBDI連動  
- 銀行株の日米長期金利スプレッド連動  
  
これらは銘柄またはセクター個別性を考慮しないと、命名上は同じパターンでも実運用での再現性が大きく劣化する。穴12は **「同じ命名のパターンでも、適用銘柄群を絞らないと再現性が壊れる」** という見落としを指す。  
  
## 対応方針  
  
**命名規則は変更しない**。パターンに以下のメタデータを持たせて適用範囲を別軸で管理する。  
  
| 項目 | 内容 |  
|---|---|  
| sector_filter | 機能するセクター名のリスト(例: `["半導体","電気機器"]`) |  
| symbol_whitelist | 機能する銘柄コードのリスト(例: `["8035","6920"]`) |  
| symbol_blacklist | 適用しない銘柄コードのリスト |  
  
3項目とも空なら制限なし(全銘柄に適用可)とする。whitelist と blacklist が両方指定された場合、whitelist が優先される。  
  
## 類似局面検索でのフィルタ適用  
  
第37章 Active Priority Set からの類似局面検索(R-37-08)はマッチング前にこのフィルタを適用する。具体的には:  
  
1. 現在の検討対象銘柄を取得  
2. Pattern Library の各パターンについて sector_filter / symbol_whitelist / symbol_blacklist を確認  
3. 対象銘柄が適用範囲外のパターンは類似局面候補から除外  
4. 残ったパターンに対してのみ局面類似度を計算  
  
これによりセクター/銘柄個別性のあるパターンが、無関係な銘柄に誤って適用されることを防ぐ。  
  
## Pattern Research Agent の拡張  
  
第21章 R-21-15 に基づき、Pattern Research Agent は以下の研究機能を持つ:  
  
- **セクター別成績集計**: 既存パターンをセクター別に分割集計し、セクターによって成績が大きく異なる場合は sector_filter 追加を提案  
- **銘柄連動パターンの発見**: 特定銘柄群でのみ高勝率となるパターン候補を抽出し、symbol_whitelist 付きパターンとして提案  
- **絞込みフィルタの修正提案**: 既存パターンのフィルタが過大/過小である兆候を検知して提案  
  
すべて提案までとし、最終採用はユーザー承認制(R-21-13と同等)。  
  
## なぜ命名規則を変更しないか  
  
- パターン命名は再現性確認の中核(第31章 Pattern Store品質管理)であり、頻繁に変更すべきでない  
- 命名にセクター/銘柄を含めると、同一の値動きパターンが銘柄ごとに別パターンとして登録され、Pattern Library が肥大化する  
- セクター/銘柄絞り込みは「同じパターンの適用範囲」を定義する別軸であり、命名規則とは直交する  
- バージョン管理(R-31-04)を複雑にしない
