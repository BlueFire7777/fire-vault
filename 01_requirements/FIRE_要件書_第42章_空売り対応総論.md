# 第42章: 空売り対応総論

本章は v3.4 で新設した空売り対応に関する横断的なまとめ章。各章への追記内容を一覧化し、実装の道標とする。

## 設計の根本原則

1. **空売りは補助レーン (買いが主戦略)**: 第03章で買いレーン1/2 を主戦略、空売りレーン3 (S1〜S5) を補助に位置付け
2. **段階導入**: M=0〜M+1 Paper Live のみ → M+2 で 100株実弾 → M+3 以降で買いの半分リスク
3. **direction 軸の独立**: Pattern Store / 類似度検索 / 階層管理 / 状態遷移すべて direction (long/short) で独立
4. **半自動主軸維持**: 空売りでも自動発注なし、Computer Use なし、強制クローズ自動化なし
5. **デイトレ専用**: 信用売建持越し禁止 (15:10までに返済買)、14:00 以降新規ショート禁止
6. **逆日歩リスク回避**: 一般信用売り or いちにち信用売りのみ使用、制度信用は原則使わない
7. **主役テーマ逆張り禁止**: 安易な逆張りショートを禁止

## 重要な現実認識

| 項目 | 結論 |
|---|---|
| 空売り価格規制 (51単元ルール) | FIRE は 100〜200株運用なので適用除外 |
| 空売り価格規制トリガー (10%下落) | 抵触銘柄は警戒。JPX サイトで日次取得 |
| 逆日歩 | 一般信用 + デイトレなので原理上発生しない |
| 売禁 | JPX + 楽天証券のリストを併用、Phase 1 は手動チェック含む |
| 一般信用売り在庫 | 楽天証券固有データ、Phase 1 は手動チェック |

## データソース3層

- **Layer A (J-Quants 自動)**: 貸借区分、信用残、業種別空売り比率、空売り残高報告、日々公表信用残高
- **Layer B (JPX スクレイピング)**: 信用取引規制銘柄、空売り規制トリガー抵触銘柄
- **Layer C (楽天証券固有 / 手動)**: 一般信用売り在庫、楽天証券社内規制、いちにち信用対象
