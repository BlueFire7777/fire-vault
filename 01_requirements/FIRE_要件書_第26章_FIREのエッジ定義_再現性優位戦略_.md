# 第26章: FIREのエッジ定義(再現性優位戦略)

## 基本定義

FIREの中核的なエッジは、**成功再現性・勝ちパターンの蓄積・現在局面との類似照合・パターン別勝率と期待値の活用**にある。

FIREは、未来を曖昧に予測するシステムではなく、過去およびリアルタイム相場から再現性の高い勝ちパターンを研究・蓄積し、現在局面がそのパターンにどれだけ近いかを評価し、**再現性が高い局面だけを執行する**システムとして定義する。

## FIREがエッジとしないもの

以下は単独のエッジとしては扱わない(補助情報または一部特徴量として扱うことはある)。
- AIの文章生成力
- ブラックボックスな未来予測
- 初動という概念そのもの
- 決算というイベントそのもの
- SNSや有名トレーダーの発言の単純追随
- コピートレードそのもの
- 板の微小な歪みを抜く高頻度戦略

## エッジを構成する5要素

| 要素 | 内容 |
|---|---|
| 1. 成功再現性 | 類似局面が過去にどれだけ存在したか / その局面でどの程度勝てていたか / その優位性が最近も維持されているか |
| 2. パターン蓄積 | 勝ちパターンをPattern Libraryに蓄積 / 新規候補をCandidate Poolに / 劣化をDEATH NOTEに / 再検証をREHAB Queueに |
| 3. 類似照合 | 現在局面を過去パターンに照合 / 類似度スコア算出 / 類似件数・レジーム・時間帯整合を確認 |
| 4. 勝率と期待値 | 勝率・平均利益・平均損失・平均RR・期待値・最大逆行・最大DDをパターン単位で管理 |
| 5. 執行品質 | シグナルが良くても約定品質が悪ければ優位性は崩れる。執行品質ゲートもエッジの一部 |

## 再現性の定義

FIREにおける再現性とは、**現在局面と類似した過去局面において、一定以上の件数と一定以上の期待値をもって、同様の方向性・利確到達・損切抑制が確認されている状態**を指す。

**方針**:
- 類似件数が少なすぎるものは再現性が高いとはみなさない
- 勝率だけでなく期待値で評価する
- 過去に強くても直近で劣化していれば再現性低下とみなす

## パターン別勝率・期待値の管理

FIREは**銘柄単位ではなく、パターン単位**で勝率と期待値を管理する。

**各パターンで保存・参照する指標**:
類似件数 / 勝率 / 平均利益 / 平均損失 / 平均RR / 期待値 / 最大逆行 / 平均保有時間 / レジーム別成績 / 直近成績 / 執行品質スコア

**重要方針**:
- 勝率が高くても平均利益が小さすぎるなら評価を下げる
- 勝率が中程度でも期待値が高ければ有力候補とする
- 類似件数が少ない場合は参考情報に留める

## 再現性スコアの構成要素

**再現性スコア** = 類似度 + 類似件数 + 勝率 + 平均RR + 期待値 + レジーム適合度 + 直近の有効性 + 執行品質 + リスク管理適合度 の合成

## 初動・決算・テーマの位置づけ

| 概念 | FIREでの扱い |
|---|---|
| 初動 | 有力な局面候補ではある。ただし「初動だから強い」とは決めつけない。再現性が確認された初動パターンのみ採用 |
| 決算 | 高品質材料の代表例。ただし決算というイベントそのものではなく、決算後のパターン再現性を重視 |
| テーマ | 主役テーマへの資金流入は重要。ただしテーマ性だけでは執行しない。既存パターンとの整合を確認 |

## AIの役割

| 区分 | 内容 |
|---|---|
| AIにやらせること | 特徴量生成 / 類似局面検索 / パターン要約 / 勝率・RR・期待値の説明 / 提案書生成 / 劣化検知 / 改善提案 |
| AIにやらせすぎないこと | 根拠不明なブラックボックス予測 / 説明不能な売買判断 / 勝手なルール変更 / 勝手なパターン削除 |

## ユーザーへの説明責任

FIREは、ユーザーが「なぜこの局面で打つのか」を理解できるようにしなければならない。執行候補や評価提案では少なくとも以下を示す。

**類似パターン名 / 類似件数 / 勝率 / 平均RR / 失敗しやすい条件 / 現在局面との一致点 / 執行上の注意点**

これにより、FIREは**雰囲気で提案するAIではなく、再現性を根拠に提案するAI**となる。

## 最終方針(一文要約)

FIREの中核エッジは、成功再現性・勝ちパターンの蓄積・現在局面との類似照合・パターン別勝率と期待値・劣化パターンの排除にある。すなわちFIREは、過去とリアルタイムから勝ちパターンを研究し、それを蓄積し、現在局面を照合し、再現性が高い局面だけを執行し、劣化したパターンを切り、改善提案を継続することで優位性を作るシステムとする。
