# 第17章: 執行品質設計(穴2対策)

FIREが壊れやすいのは予測が外れる時より、**執行が雑な時**である。日本株特有の寄り付きギャップ、特買い・特売り、板が一瞬で飛ぶ現象、値嵩株のスリッページ、部分約定、後場寄りの歪みは、シグナル精度と約定品質を別物として設計しないと優位性を崩す。

FIREは「高精度な予測システム」ではなく、**「約定品質を守る執行システム込みの売買システム」**として設計する。

## 執行品質の基本方針

| ルール | 内容 |
|---|---|
| 成行乱用禁止 | 通常時の新規エントリーで成行を標準にしない |
| 寄り直後制限 | 9:00〜9:05は原則「観察優先」、9:05以降または初動安定後に執行 |
| 特買い・特売り禁止 | 特買い・特売り中は新規発注禁止 |
| 価格乖離上限 | シグナル時価格と実発注価格のズレに上限(+0.3% / 値嵩+20円超で無効) |
| 部分約定前提 | 100株全部入る前提で考えない。半端な約定でも逆指値を必ずセット |
| 強制クローズ前倒し | 15:00〜15:05で整理優先、15:08以降は強制モード。15:10は「完了期限」 |

## Execution Quality Gate

候補スコアが高くても、**Execution Quality Gate がNGなら見送り**とする。ゲートで確認する項目:

板の厚さ / スプレッド / 直近出来高 / 部分約定リスク / 値飛びリスク / 値嵩株としての滑りやすさ

## 執行ルール詳細

| フェーズ | ルール |
|---|---|
| 新規エントリー | 標準は指値。成行は原則使わない。価格乖離一定以上なら見送り。特買い・特売り中は発注禁止 |
| 利確 | 基本は指値。利確1は事前注文可。利確2はストップ引き上げも可 |
| 損切 | 逆指値必須。ただし滑りはログ保存。想定より大きく滑る銘柄は次回減点 |
| 強制クローズ | 前倒しで段階的に整理。最終時刻は余裕を持つ。クローズ不能時は即警告 |

## 執行品質スコアの段階的監視(実弾スリッページ管理)

実弾運用における約定のたびに `fill_quality_score` を算出し、想定スリッページとの乖離を以下の**3段階**で監視・制御する。

- **注意レベル**: 直近5トレードのスリッページ中央値が想定の**2.0倍超** → LINE警告のみ
- **警戒レベル**: 直近10トレードのスリッページ中央値が想定の**2.5倍超** → 当該日の新規エントリーリスク半減
- **停止レベル**: 直近10トレードのスリッページ中央値が想定の**3.0倍超** → 当該日の新規エントリー即時停止・ユーザー確認

※ 運用開始30営業日以降は、直近30日の実測スリッページ中央値を基準として監視する。

## Dashboardへの反映

執行品質を可視化しないと「戦略は良いのに約定で負けてる」が見えない。以下をDashboardに表示:

想定価格 vs 実約定価格 / 乖離率 / 部分約定率 / 未約定率 / 損切スリッページ / 利確スリッページ / 強制クローズ率 / 執行品質スコア

## 空売り執行品質ゲート追加項目 (v3.4 追加)

買い側 (R-17-07) のチェック項目に加え、空売り発注時は以下も Execution Quality Gate でチェックする:

- **貸借区分**: 貸借可であること (J-Quants `/listed/info` で取得)
- **売禁チェック**: 売禁銘柄でないこと (JPX 信用取引規制銘柄リスト)
- **空売り価格規制トリガー抵触チェック**: 当日基準価格から-10%以上下落していないこと (JPX 空売り規制ページ)
- **一般信用売り在庫**: 楽天証券で在庫ありであること (Phase 1 は手動チェック)
- **踏み上げ初動チェック**: 直近5分の出来高急増 + 価格急上昇が無いこと

これらのいずれかが NG の場合、スコアに関係なく **見送り** (R-10-09 とも一致)。
