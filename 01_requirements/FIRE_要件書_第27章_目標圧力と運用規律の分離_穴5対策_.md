# 第27章: 目標圧力と運用規律の分離(穴5対策)

## 穴5の本質

資産目標(現在780万円 → 2年で3,000万円)は明確だが、**目標未達を理由に運用規律が崩れることが最大のリスク**。具体的には、ロットを上げすぎる・無理に毎日打つ・準主戦略を主戦略並みに扱う・paper段階なのにfull autoに急ぐ・DDを許容しすぎる、といった目標ドリブンの過剰攻撃が起きる。

**結論**: FIREは目標進捗を追うべき。ただし、**目標未達を理由にリスクを自動で上げてはいけない**。

## 固定原則

| 原則 | 内容 |
|---|---|
| 目標は監視する | 乖離は可視化する |
| 自動攻撃化禁止 | 乖離を埋めるために自動で攻撃化しない |
| リスク引き上げ条件 | 再現性と相場環境が揃った時だけ |
| 目標未達時の対応順序 | 第1対応: 精度改善 → 第2対応: 頻度改善 → 第3対応: 限定的なリスク調整 |

## 禁止事項(自動攻撃化の具体例)

以下は目標乖離を理由にしても実施してはならない。
- 乖離率が大きいからロット自動増加
- 乖離率が大きいから候補採用基準を下げる
- 乖離率が大きいから準主戦略を主戦略化する
- 乖離率が大きいから停止条件を緩める

## リスク量を変える正しい条件

ロットやリスク量を変える条件は**目標乖離ではなく以下の相場条件**で決める。

| 条件 | 内容 |
|---|---|
| Pattern Score | 再現性スコアが高い |
| Regime Fit | 地合いが追い風 |
| Execution Quality | 約定品質が良好 |
| Recent Stability | 直近の成績が安定している |
| System Health | システム状態が正常 |

**要約**: 「足りないから上げる」ではなく「勝ちやすいから上げる」。

## 目標未達時のEvaluation Agent提案の方向性

目標未達時の警告は出す。ただし提案の方向性は以下の順序に従う。

1. 今月は頻度不足
2. 主戦略Aの再現性低下
3. 候補抽出が狭すぎる
4. 約定品質悪化
5. ロット不足ではなく、精度側に問題あり

## 攻める日 / 守る日の概念

攻める日と守る日は、**目標乖離ではなく市場条件**で決める。

| 区分 | 条件 |
|---|---|
| 攻める日 | 主戦略パターンの再現性が高い / 地合いが追い風 / 主役テーマ明確 / 執行品質良好 / System health良好 |
| 守る日 | 再現性低下 / 地合い不安定 / 候補不足 / 執行品質悪化 / データ欠損気味 |

## Dashboardで必要な表示

進捗系と運用状態系を**並列表示**することで「遅れてるから無理に行く」を防ぐ。

- **進捗系**: 現在資産 / 理想進捗との差 / 乖離率 / 今週必要進捗 / 今月必要進捗
- **運用状態系**: 現在のリスクモード / ロットモード / 停止状態 / 攻めてよい相場か

## 穴5の思想(一文要約)

FIREは資産目標を追うが、目標未達を埋めるために無理に攻めるシステムではない。目標乖離は改善課題の発見に使い、攻撃性の引き上げは再現性・地合い・執行品質が揃った場合に限定する。
