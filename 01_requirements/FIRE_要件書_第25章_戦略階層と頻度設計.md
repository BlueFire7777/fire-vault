# 第25章: 戦略階層と頻度設計

## 背景: 頻度と勝率のバランス

主戦略を厳しく絞りすぎると「安全だけど全然打てない」システムになる。FIREは**期待値×頻度**で最適化すべきであり、「毎日必ず打つ」ではなく「**打てる日を増やす**」設計にする。

## 3レーン構造

| レーン | 区分 | 内容 | 用途 |
|---|---|---|---|
| レーン1 | Primary Edge(最優先主戦略) | 高品質材料の初動 / 主役テーマ連動の値嵩株前場初動 | 最も期待値が高い本命 |
| レーン2 | Secondary Edge(準主戦略) | 前日強かった主役株の押し目再加速 / セクター主役続伸初押し / 前場ブレイク再試行 / 材料株2日目初押し | 主戦略不在日の補完 |
| レーン3 | Research Edge(研究/paper限定) | 準スキャル / 新規パターン候補 / 条件やや弱いが監視価値ありのもの | 本番主軸にしない |

## 運用ルール

- 本命がある日は本命(レーン1)を打つ
- 本命がない日は準主戦略(レーン2)に降りる
- それでも基準未満ならノートレでよい
- ただし、ノートレばかりにならないように戦略層を設計する
- 毎日無理に打たない。ただし主戦略不在時は準主戦略まで降りて候補を探す

## 頻度目標(理想レンジ)

| 指標 | 目標 |
|---|---|
| 1日あたりのトレード数 | 0〜3件 |
| 週あたりのトレード数 | 5〜10件(理想レンジ) |
| 頻度不足の検知 | 2週間連続で極端に少ない場合、フィルター厳しすぎを点検。Evaluation Agentが改善提案 |

**重要**: トレード頻度不足も**劣化として監視対象**にする。

## Dashboardで監視する頻度関連指標

日次トレード件数 / 週次トレード件数 / 主戦略別件数 / 準主戦略別件数 / ノートレ日数 / 候補不足日数 / フィルター除外率

## 決算シーズンの特別ルール

| 項目 | 内容 |
|---|---|
| 材料母集団 | 広く取る(決算期は候補が増える) |
| 実行候補数 | 増やさない(1〜3件固定) |
| フィルター | 通常時より厳格化 |
| 優先条件 | 値嵩株 / 流動性十分 / 主役テーマ性あり / サプライズ度が高い / 還元強化も伴う / 寄り後の形が明確 / 前場でRRが作れる |
| 除外条件 | 決算は良いがすでに織り込み済み / 板が薄い / ギャップが大きすぎる / 値動きが荒すぎる / 執行品質が悪い |
| 決算またぎ | 原則主戦略にしない |
| 優先基準 | 「良い決算」より「市場が強く再評価する決算」を優先 |
