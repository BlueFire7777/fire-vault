# 第38章: 運用モード / 休止 / 旅行モード方針(v3.3: 緊急ポジション整理含む)

## v3.3 での変更

v3.2 では「強制クローズは独立 Cron + Playwright で自動化」だったが、v3.3 では **LINE 緊急アラートで人間対応** に変更。本章は運用モード全般 + 緊急ポジション整理の取り扱いを扱う。

## 基本思想

FIRE は、稼働を開始しやすいだけでなく**安全に止めやすい**ことも重要な品質とする。

## 通常運用モードの定義(5段階)

| モード | 新規発注指示(LINE) | 相場監視 | 候補抽出 | Paper Live | 人間発注 |
|---|---|---|---|---|---|
| off | 送信しない | しない | しない | しない | 不可 |
| observe_only | 送信しない | する | する | 任意 | 不可 |
| paper_live | 送信しない(仮想のみ) | する | する | する | 不可 |
| **live_advisory (v3.3改訂)** | **送信する** | する | する | する | **Fujiwara さんが手動発注** |
| full_auto(将来) | 送信する | する | する | する | 自動(v3.3 時点では未実装) |

## 特別モードの定義(3種)

| モード | 用途 | 実体 |
|---|---|---|
| travel_mode | 旅行・長時間不在時の安全モード | observe_only または off |
| maintenance_mode | システム保守・設定変更 | 発注指示停止・監視継続 |
| emergency_stop | 異常時の緊急停止 | 全通知停止・即時クローズ促し |

## 🚨 緊急ポジション整理(v3.3 第6章とも連動)

### 目的

Fujiwara さんが本業中でも 15:10 までに確実に全信用建玉をクローズできるよう、段階的にアラートを送る。

### アラート時刻と内容

| 時刻 | 区分 | 通知内容 |
|---|---|---|
| 14:45 | 第一次通知 | 建玉残 X 件、15:10までにクローズしてください |
| 14:55 | 第二次通知 | 建玉残 X 件、あと15分 |
| 15:05 | 第三次通知(最重要) | ⚠️ 残り5分、即クローズを |
| 15:10 | 期限到達通知 | 強制クローズ期限到達。建玉残 X 件 |
| 15:15 | 最終確認 | 15:10時点でクローズ完了を確認してください |

### 実装要件

- **独立性**: 緊急ポジション整理アラートは OpenClaw 障害時でも稼働する(macOS 標準 Cron から起動、第23章参照)
- **確認機能**: Fujiwara さんがクローズ完了を LINE 返信 or Dashboard チェックすると、FIRE 側のログに反映
- **未完了警告**: 15:30 時点で Fujiwara さん確認がない場合、LINE で繰り返し警告

### 運用ルール補強

**建玉持越しリスクを下げる運用ルール**:

- 原則、**前場(9:00〜11:30)のみ新規エントリー**
- **後場(12:30〜15:30)は既存建玉の管理のみ、新規エントリー禁止**
- **11:30 前場引け時点で建玉ゼロを目標**
- Fujiwara さんが 15:00〜15:10 の対応困難が予測される日(重要会議等)は、**前場引けで全クローズ**
- Google Calendar または朝一言通知で当日スケジュールを共有、FIRE が「今日は後場対応困難そうです」と判断して前場引け推奨

## travel_mode の詳細(v3.3)

| 項目 | 内容 |
|---|---|
| 新規発注指示 LINE | 禁止 |
| 実運用 | 停止 |
| 既存建玉 | 原則持たない(移行前に整理) |
| 緊急ポジション整理機能 | 維持 |
| 監視継続 | 選択可 |
| Paper Live 継続 | 選択可 |
| Pattern Research / Evaluation / 夜間集計 | 継続してよい |

### travel_mode 開始時の安全確認

- 建玉が残っていないか
- 未約定注文が残っていないか(※手動発注なので Fujiwara さん側の注文管理も要確認)
- live_advisory から安全に切り替わったか
- LINE / Dashboard に移行完了通知

### travel_mode 解除時の安全確認

- 楽天証券約定メール取込の復旧確認
- 市場データ取得正常
- LINE 通知正常
- Dashboard 正常
- OpenClaw 正常

**方針**: travel_mode 解除後、直ちに live_advisory に戻すことを**標準としない**。まず observe_only または paper_live で再開し、段階的に上げる。

## モード切替方法

| ツール | できること |
|---|---|
| Dashboard | 運用モード変更 / travel_mode 開始・解除 / 休止期間設定 / 緊急停止 / 再開 |
| LINE | 停止 / 監視のみ / paperに変更 / 旅行モード開始・解除 / 期間指定停止 / システム状態確認 |

## 夜間処理との関係

運用モードが休止中でも、以下の夜間処理は継続してよい:

Pattern Research / Evaluation Agent 分析 / Dashboard 更新 / レポート生成準備 / データ整備 / システム疎通確認
