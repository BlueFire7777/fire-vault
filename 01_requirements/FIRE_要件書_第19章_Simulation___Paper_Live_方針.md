# 第19章: Simulation / Paper Live 方針

FIREは開発完了直後にいきなり実資金で本番運用を開始してはならない。本番移行前に**Simulation ModeおよびPaper Live Modeによる十分な検証**を行い、戦略精度・執行品質・停止機能・強制クローズ機能・シミュレーション精度を確認することを必須とする。

## 検証モードの全体構成(5段階)

| モード | 内容 |
|---|---|
| 1. Backtest Mode | 過去データを用いてルールベース検証。戦略の大まかな有効性、候補抽出、エッジ別成績、損益傾向を確認 |
| 2. Replay Simulation Mode | 過去の特定日を時系列で再生。当時利用可能だった情報のみを使ってFIREが判断した場合の成績を確認。後知恵バイアスを減らした検証 |
| 3. Paper Live Mode | リアルタイム相場で稼働。実発注は行わず、仮想売買を記録。**本番移行前の最重要検証モード** |
| 4. Semi Auto Mode | 実相場で稼働。発注前にユーザー承認を必要とする |
| 5. Full Auto Mode | 実相場で稼働。条件一致で自動発注・自動利確・損切・停止・強制クローズ |

**Stage飛ばし禁止**。この順序を必ず守る。

## Paper Live Mode の役割

Paper Live Modeで行うこと:
朝レポート生成 / 候補抽出 / 実行候補1〜3銘柄への厳選 / 条件監視 / エントリーしたと仮定 / 利確したと仮定 / 損切したと仮定 / 強制クローズしたと仮定 / 仮想損益・仮想建玉・仮想余力を記録 / LINE通知も本番同様に出す

実際の市場データを使い、実発注はしない。自動停止や強制クローズの挙動も検証対象とする。

## 仮想売買の対象

**仮想執行対象**: 新規買い / 利確 / 損切 / 強制クローズ / 注文取消 / 注文失効 / 建玉管理 / 仮想余力管理

**管理する仮想情報**: 仮想建玉 / 仮想平均建値 / 仮想評価損益 / 仮想実現損益 / 仮想余力 / 仮想ロット消費状況 / 仮想停止状態

## シミュレーション約定モデル(3段階)

| モデル | 内容 |
|---|---|
| A. 理想約定モデル | シグナル価格でそのまま約定したと仮定。最も楽観的 |
| B. 現実寄り約定モデル | スプレッド・価格乖離・約定遅延・部分約定を加味。**標準精度確認用** |
| C. 厳しめ約定モデル | スリッページをやや悪化させる。楽観バイアスが大きくないか確認 |

## Simulation Accuracy の定義と主要指標

Simulation Accuracyとは、仮想売買結果が現実の執行条件にどれだけ近いかを示す指標。

| 指標カテゴリ | 指標 |
|---|---|
| 収益系 | 仮想日次/週次/月次/累積損益 |
| 精度系 | 仮想約定率、仮想利確到達率、仮想損切率、仮想強制クローズ率 |
| 乖離系 | 理想約定と現実寄り約定の差、想定スリッページ、想定部分約定率、想定未約定率、想定損切滑り |
| 信頼性系 | Simulation Confidence Score / Optimistic Bias Score / Realistic Fill Score |

## 本番移行前の必須条件

FIREは以下を満たすまでSemi Auto Mode以上へ移行してはならない。

| 条件 | 内容 |
|---|---|
| サンプル数 | Paper Live Modeを20営業日以上、または50トレード以上継続 |
| 主戦略の期待値 | 主戦略勝率・平均損益・平均RR・最大ドローダウンが基準以上 |
| 停止機能 | 自動停止が正常に機能すること |
| クローズ機能 | 強制クローズが正常に機能すること(成功率100%) |
| Simulation精度 | シミュレーションが過度に楽観的でないこと(Optimistic Bias Scoreが基準以内) |

原則: **「収益が出ていることだけを理由に本番移行してはならない」**。
